# 开发日志 — 2026-06-28

## 今日完成

- **图标源更换**：Android 启动器图标源从 `pwa-512.svg` 改为 `public/moon.png`（新月 PNG）
  - 更新 `scripts/generate-icons.mjs` 从 PNG 读取并裁剪生成 5 种密度 mipmap
  - 重新生成 `mipmap-{mdpi,hdpi,xhdpi,xxhdpi,xxxhdpi}/ic_launcher.png`
- **MainActivity 关键修复**：`registerForActivityResult` 移到 `super.onCreate()` 之前
  - ComponentActivity 要求在 `super.onCreate()` 前注册 ActivityResultLauncher
  - 修复前可能在部分设备上抛出 `IllegalStateException`
- **文档更新**：
  - `CLAUDE.md` 图标部分更新为 `moon.png` 作为源文件
  - `docs/需求规格.md` 安全部分移除过时的 Firebase 描述，改为 IndexedDB 本地存储
- **APK 重建**：使用 Gradle 8.2 构建签名 APK（967KB），嵌入新月图标
- **APK 离线化**：PWA 资源打包进 APK（1.6MB），使用 WebViewAssetLoader 本地加载
  - 添加 `androidx.webkit:webkit` 依赖
  - 重写 `MainActivity.java`：移除远程 URL，改用 WebViewAssetLoader 以 `https://appassets.androidplatform.net` 域名加载本地资源
  - 自定义 PathHandler 处理 SPA 路由回退（`/settings`、`/note/abc` 等 → `index.html`）
  - 创建 `scripts/copy-dist-to-apk.mjs` 构建脚本（dist/ → Android assets）
  - 新增 `npm run build:apk` 命令（`vite build --base=/` + 复制脚本）
- **存储保护修复**：
  - 增强 `useStorageInfo` hook：新增 `requestPersist()` 主动申请、`refresh()` 重新查询、`loading` 状态、页面可见性监听自动刷新
  - `SettingsPage` 新增「请求存储保护」按钮，点击后调用 `navigator.storage.persist()` 并显示结果反馈
