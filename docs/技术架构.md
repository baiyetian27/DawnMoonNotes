# 曦月笔记 — 技术架构

## 整体架构

```
┌─────────────────────────────────────────────┐
│                PWA 前端应用                   │
│         React 19 + TypeScript + Vite         │
│  ┌──────────┬──────────┬──────────────┐     │
│  │ 页面组件  │ 共享组件  │   Hooks       │     │
│  ├──────────┼──────────┼──────────────┤     │
│  │ TipTap   │ Tailwind │ Force Graph  │     │
│  │ 编辑器   │ 样式     │ 知识图谱     │     │
│  ├──────────┴──────────┴──────────────┤     │
│  │     Dexie.js (IndexedDB)           │     │
│  │     本地数据库 + 文件存储           │     │
│  └────────────────────────────────────┘     │
└─────────────────────────────────────────────┘
```

纯本地 PWA 应用，无后端服务器，无云数据库。所有数据存储在浏览器 IndexedDB 中。

## 技术选型理由

### 前端框架：React 19 + TypeScript
- **React**：生态最丰富，社区最大，PWA 支持好
- **TypeScript**：类型安全，减少运行时错误
- **Vite**：开发服务器秒启动，构建快

### PWA：vite-plugin-pwa + Workbox
- 将网页应用安装到手机桌面
- Service Worker 实现离线缓存
- Workbox 提供成熟的缓存策略

### 富文本编辑器：TipTap
- 基于 ProseMirror，稳定成熟
- 插件式架构，易于扩展
- 支持自定义扩展（双向链接 `@` 提及）
- 输出 JSON 格式，适合存入 IndexedDB
- 内置表格、任务列表、图片支持

### UI 样式：Tailwind CSS v4
- 原子化 CSS，开发速度快
- 自定义主题配置（紫色系）
- 深色模式支持好
- 响应式设计方便

### 本地数据库：Dexie.js
- IndexedDB 的事实标准封装库，API 简洁
- Promise 风格，完美适配 async/await
- 内置 TypeScript 类型支持
- 体积极小（~15KB gzipped）
- 支持复合索引和复杂查询

### 知识图谱：react-force-graph-2d
- 基于 D3.js 力导向算法
- React 原生封装
- 支持节点点击、缩放、拖拽
- 性能好，百级节点流畅

### 图标：Lucide Icons
- 开源线性图标库
- Tree-shaking，按需加载
- 科技感风格匹配

### 截图分享：html2canvas
- 将 DOM 元素渲染为 Canvas，导出 PNG
- 配合 Web Share API 实现分享到微信/钉钉等

### 托管：Vercel
- 免费 100GB 流量/月
- 自动 HTTPS
- Git 推送自动部署
- 全球 CDN 加速

## 数据流

```
用户操作 → React组件 → Service层 → Dexie.js → IndexedDB
                ↑                        ↓
                └── 查询返回数据 ←─────────┘
```

- 组件直接调用 Service 层函数
- Service 层封装 Dexie.js 操作
- 数据存储在浏览器 IndexedDB 中
- 无网络请求，所有操作即时响应
