# 曦月笔记 — 数据库设计

## 数据库：IndexedDB（通过 Dexie.js 封装）

所有数据存储在浏览器本地，无云端数据库。单用户模式，无需用户 ID 字段。

---

## 数据表结构

### `notes` — 笔记

```typescript
interface Note {
  id?: number;             // 自增主键
  title: string;           // 主标题（可为空）
  subtitle: string;        // 副标题（可为空）
  content: object;         // TipTap 编辑器 JSON
  plainText: string;       // 正文纯文本（用于搜索）
  tags: number[];          // 关联标签 ID
  isXizhu: boolean;        // 是否曦筑（置顶）
  isLocked: boolean;       // 是否加锁
  outgoingLinks: number[]; // 链接到的笔记 ID
  createdAt: number;       // Unix 时间戳（毫秒）
  updatedAt: number;       // 最后更新时间戳
}
```

**索引**：
- `++id` 自增主键
- `title`
- `plainText`
- `tags`
- `isXizhu`
- `isLocked`
- `updatedAt`

### `tags` — 标签

```typescript
interface Tag {
  id?: number;
  name: string;            // 标签名（最长 10 字）
  color: string;           // red | orange | yellow | green | blue | purple | default
  createdAt: number;
}
```

### `links` — 双向链接

```typescript
interface Link {
  id?: number;
  sourceNoteId: number;    // 来源笔记 ID
  targetNoteId: number;    // 目标笔记 ID
  sourceTitle: string;     // 来源笔记标题（冗余展示）
  targetTitle: string;     // 目标笔记标题（冗余展示）
  createdAt: number;
}
```

**索引**：`sourceNoteId`、`targetNoteId`

### `settings` — 应用设置

```typescript
interface AppSettings {
  id?: number;
  autoXizhuTagIds: number[];  // 智能曦筑标签（最多 3 个）
}
```

### `images` — 图片存储

```typescript
interface ImageRecord {
  id?: number;
  noteId: number;          // 所属笔记 ID
  data: string;            // base64 data URL
  filename: string;        // 原始文件名
  createdAt: number;
}
```

**索引**：`noteId`

---

## 搜索策略

使用 Dexie.js 的客户端过滤 + 字符串包含匹配：
- 搜索 `title`、`subtitle`、`plainText` 字段
- 使用 `String.includes()` 做子串匹配（不区分大小写）
- 后期可接入 `lunr.js` 等客户端全文搜索引擎提升搜索体验

---

## 数据迁移

通过 `src/services/migration.ts` 实现导出/导入：
- 导出：所有表数据序列化为单个 JSON 文件
- 导入：清空现有数据，从 JSON 恢复
- 保持 ID 映射关系以确保链接完整性

详见 `docs/数据迁移指南.md`
